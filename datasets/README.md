# Datasets

This directory contains recorded mission runs.

A dataset represents a single execution instance of:

- Simulation run
OR
- Hardware run

Each dataset must be self-contained and reproducible.

---

# 1. Directory Structure


datasets/
<run_id>/
run_metadata.json
run.mcap
metrics.json (optional)
plots/ (optional)
schemas/
run_metadata.schema.json


Each `<run_id>` directory corresponds to one execution.

Recommended naming convention:

YYYYMMDD_HHMMSS_<scenario>_<git_short_hash>

Example:

20260222_031500_mars_flat_a83f2c1

---

# 2. Required Files

Every dataset directory must contain:

## run_metadata.json

Must validate against:

`datasets/schemas/run_metadata.schema.json`

Metadata defines:

- Git commit
- Physics engine
- Parameters used
- Topics recorded
- Deterministic seed
- Config files
- Integrity hashes

If metadata is missing or invalid,
the dataset is not usable.

---

## run.mcap

Primary ROS 2 bag (MCAP format).

Must contain:

- /tf
- /joint_states
- /odometry/filtered
- /wheel/odometry_raw
- /imu/data
- /cmd_vel_safe
- /mission/state
- Any scenario-required topics

MCAP is authoritative data.
CSV exports are secondary artifacts.

---

## metrics.json 

Generated by simulation regression or analysis.

May contain:

- slip statistics
- mission completion status
- force balance metrics
- acceptance results

Metrics must be derived, not manually edited!

---

# 3. Data Integrity Rules

Datasets are immutable.

Once created:

- Do not modify run.mcap
- Do not modify metadata
- Do not overwrite metrics

If corrections are needed,
generate a new run_id.

Reproducibility depends on immutability.

---

# 4. Simulation or Hardware

mode = "simulation"
or
mode = "hardware"

Simulation runs must include:

- seed
- physics configuration
- real_time_factor
- deterministic flag

Hardware runs must include:

- sensor calibration references
- firmware version
- hardware identifiers

---

# 5. Promotion Policy

Datasets used for calibration must:

- Be marked usable in dataset_index.json
- Pass schema validation
- Contain complete topic set

Datasets used for regression must:

- Be deterministic
- Use fixed scenario parameters
- Be reproducible from commit

---

# 6. Relationship to Analysis

The analysis framework:

`analysis/calibration/`

Consumes datasets from this directory.

Calibration never modifies datasets.

All derived artifacts must be written to:

`analysis/calibration/outputs/`

---

# 7. Storage Policy

Large datasets should:

- Be stored using Git LFS
OR
- Be archived externally with checksum references

The repository must not silently break due to missing bags.

Integrity hashes are mandatory for long-term archival.

---

# 8. Engineering Principle

A dataset is a frozen experiment.

If the system changes, a new dataset must be generated.
