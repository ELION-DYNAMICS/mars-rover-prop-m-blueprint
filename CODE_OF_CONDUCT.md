# Code of Conduct

## Purpose

This project exists to build high-quality, reproducible, engineering-grade robotics systems.

We value rigor, discipline, and respectful collaboration.

Participation in this repository means agreeing to uphold the standards defined below.

---

## Our Standards

### 1. Professional Conduct

All interactions must remain:

- Respectful
- Constructive
- Technically focused
- Free from harassment or discrimination

We critique ideas and implementations - never people.

---

### 2. Engineering Integrity

We do not:

- Submit untested code
- Merge undocumented features
- Bypass CI checks
- Obfuscate performance regressions
- Fabricate results or benchmarks

All contributions must be technically honest and reproducible.

---

### 3. Communication Guidelines

When reviewing code or designs:

- Be precise
- Provide actionable feedback
- Reference metrics or documentation
- Avoid personal tone

Strong technical disagreement is welcome. Personal attacks are not.

---

### 4. Inclusivity

We are committed to maintaining a workspace free of discrimination based on:

- Gender
- Identity
- Nationality
- Religion
- Race
- Disability
- Age
- Background

Technical merit and professionalism are the only criteria that matter.

---

## Enforcement

Maintainers reserve the right to:

- Remove content that violates this Code
- Reject contributions that fail engineering standards
- Restrict or revoke participation for repeated violations

Serious violations may result in permanent removal from the project.

---

## Reporting

If you encounter behavior that violates this Code:

- Open a private issue to the maintainers
- Or contact the repository owner directly

All reports will be reviewed confidentially and seriously.

---

## Final Principle

This is a robotics engineering project.

We are here to build systems that could survive Mars-level constraints.

Conduct should reflect that standard.
